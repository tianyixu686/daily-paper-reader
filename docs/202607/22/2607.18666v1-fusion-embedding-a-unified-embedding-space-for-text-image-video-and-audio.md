## 摘要
一个覆盖文本、图像、视频和音频的单一嵌入空间，使得一个索引能够服务于用户可能提出的任何查询。基于视觉-语言骨干构建的嵌入模型如今在文本/图像/视频检索基准测试中领先，但完全缺乏音频能力；而音频-文本检索则由专门服务于单一模态的系统主导。我们提出了融合嵌入家族，它在冻结的视觉-语言嵌入基座（其参数从未更新）中增加了音频：第一代（fusion-embedding-1）仅在冻结的音频塔与冻结的基座之间训练一个16.4M参数的连接器；第二代（fusion-embedding-2）增加了模态门控深度适配器（44.2M参数），其分支从未在文本、图像或视频输入上执行：它们的输出与已发布基座的输出逐位相同，每次训练运行后均经过验证。由于基座已经绑定了文本、图像和视频，仅将音频与文本对齐即可实现音频-图像检索，而无需任何成对的音频-视觉训练数据。除了这一方案，我们还通过受控的负面结果（使用LLM重写训练标题、替换排行榜上更强的音频塔、加宽连接器均会降低检索性能）以及训练协议方面的发现（我们预计这些发现可迁移至任何冻结的解码器-LM嵌入骨干）来映射其设计空间。两代模型均在单张GPU上数小时内完成训练。权重、代码和评估工具包均已公开发布。

## Abstract
A single embedding space that covers text, images, video, and audio lets one index serve every query a user can pose. Embedding models built on vision-language backbones now lead text/image/video retrieval benchmarks but lack audio entirely, while audio-text retrieval is led by specialist systems that serve no other modality. We present the Fusion Embedding family, which adds audio to a frozen vision-language embedding base whose parameters are never updated: generation 1 (fusion-embedding-1) trains only a 16.4M-parameter connector between a frozen audio tower and the frozen base, and generation 2 (fusion-embedding-2) adds modality-gated deep adapters (44.2M parameters) whose branch never executes on text, image, or video inputs: their outputs are bit-for-bit those of the released base, verified after every training run. Because the base already binds text, images, and video, aligning audio to text alone makes audio-image retrieval emerge, with zero paired audio-visual training data. Alongside the recipe we map its design space with controlled negative results (rewriting training captions with an LLM, substituting a leaderboard-stronger audio tower, and widening the connector each reduce retrieval) and with training-protocol findings that we expect to transfer to any frozen decoder-LM embedding backbone. Both generations train in hours on a single GPU. Weights, code, and the evaluation harness are openly released.

### 2. 方法论：核心思想、关键技术细节
- **核心思想**：使用一个**完全冻结的视觉‑语言嵌入基座**（Qwen3‑VL‑Embedding‑2B）和一个冻结的音频塔（Qwen2.5‑Omni 的音频编码器），仅训练一个轻量的**连接器**（Generation 1）或加**模态门控深度适配器**（Generation 2）将音频映射到基座的输入空间，所有文本、图像、视频路径的计算完全不变。
- **技术细节**：
  - **Generation 1**：训练一个 **FusionResampler**（Flamingo 风格的 perceiver resampler），参数量 16.4M，内部瓶颈宽度384，包含64个可学习潜变量查询，6个预归一化块。它将音频特征帧映射为64个token，替换基座输入中的占位符token位置，然后由冻结的 LLM 进行前向传播，最终通过基座自己的最后 token 池化和 Matryoshka 截断得到嵌入。
  - **Generation 2**：在 Generation 1 基础上，为基座的每个解码器层（28层）添加**瓶颈适配器**（LayerNorm → 2048×384 → SiLU → 384×2048），共44.2M参数。适配器通过**硬门控**仅在音频输入时执行，文本/图像/视频输入时跳过，确保非音频输出的逐位不变性。适配器上投影零初始化，训练从 Generation 1 的架构开始。
  - **训练目标**：对称 InfoNCE 损失，在每个 Matryoshka 梯级（64/128/256/512/1024/1536/2048维）上计算并加权求和，加上轻量 CORAL 协方差惩罚（λ=0.05）。使用**完整语料库的冻结文本负样本库**（无陈旧问题）、**软标签**（β=0.3）和**错误负样本掩码**（余弦相似度>0.98时屏蔽）。
  - **关键预处理**：将文本嵌入进行**对角白化**（通过预计算的均值、标准差），保持 Matryoshka 截断与白化可交换；使用**本机输入协议**（基座训练时的 chat 模板格式），该效应带来 +14.5 R@10 的提升。
- **公式/算法流程**（文字说明）：
  1. 音频输入经过冻结音频塔 → 帧特征 → FusionResampler → 音讯 token（64个）插入基座输入序列。
  2. 基座（冻结 LLM）执行前向，输出以 EOS 位置后的嵌入向量。
  3. 嵌入向量经过 Matryoshka 截断（多梯级）和 L2 归一化。
  4. 损失函数为每个梯级上的对称 InfoNCE（全库负样本） + CORAL 协方差项。
  5. 仅在 Generation 2 中，每个解码器层内的适配器被门控激活，执行前向/反向。

### 6. 主要结论与发现
- **冻结基座可以高效添加音频**：仅训练 16.4M~60.6M 参数，即可获得与双塔完全微调可比的表现（AudioCaps A→T R@10 0.741/0.743，T→A R@10 0.746/0.775）。
- **零样本跨模态涌现**：从未见过 audio‑image 对，仅靠 audio‑text 对齐，在 VGGSound‑696 上达到 R@10 0.418（29倍随机），且随着 audio‑text 性能提升同步增长。
- **本机输入协议是最大杠杆**：使用与基座训练一致的 chat 模板评估，带来 +14.5 R@10 的提升（图2），此发现应推广到任何冻结 Decoder‑LM 嵌入骨干。
- **负面结果揭示真正瓶颈**：不是数据质量、音频塔或连接器大小，而是**冻结 LLM 内部处理音频 token 的能力**，Generation 2 的模态门控适配器正是为此设计，并产生一致改进（T→A 方向最显著）。
- **语料规模超过500K后全库负样本库出现拥挤问题**，需用软标签和错负掩码缓解。

### 7. 优点：方法或实验设计的亮点
- **保障存量部署**：冻结基座+门控适配器保证文本/图像/视频嵌入**逐位不变**，已索引的向量直接复用，无需重新嵌入。
- **极度高效**：单 GPU 数小时完成训练，远低于端到端微调方案（如 ONE‑PEACE 4B 参数训练）。
- **开放全面**：权重、代码、评估工具全部公开，含负样本库、精确复现配置和自动化检测脚本。
