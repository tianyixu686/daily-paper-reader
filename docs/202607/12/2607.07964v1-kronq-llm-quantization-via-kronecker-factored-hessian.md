## 摘要
后训练量化（PTQ）是一种广泛采用的压缩大型语言模型（LLM）而无需重新训练的技术。现有的二阶PTQ方法，包括GPTQ，完全基于输入激活统计构建量化目标，实际上假设所有输出通道对逐层重建目标的贡献相等。我们提出KronQ，一种通过将梯度协方差引入量化流水线来挑战这一假设的PTQ框架。在Kronecker因子化Hessian近似下，量化损失同时依赖于激活协方差和梯度协方差，KronQ在两个互补的层面上利用这一点。（1）KronQ引入了双向非相干处理，利用梯度协方差将现有的输入侧随机旋转扩展到输出维度，从而减少跨输入和输出维度的权重幅度方差。（2）KronQ推导了一种新的层间混合精度分配敏感性指标，该指标由梯度和激活Hessian迹驱动。值得注意的是，在LLaMA-3-70B上进行2位仅权重量化的情况下，虽然GPTQ和GPTAQ发散或产生退化量化（在WikiText-2上超过2000的困惑度），但KronQ实现了7.93的困惑度。

## Abstract
Post-training quantization (PTQ) is a widely adopted technique for compressing large language models (LLMs) without retraining. Existing second-order PTQ methods, including GPTQ, construct quantization objectives exclusively from input activation statistics, effectively assuming that all output channels contribute equally to the layer-wise reconstruction objective. We propose KronQ, a PTQ framework that challenges this assumption by introducing the gradient covariance into the quantization pipeline. Under the Kronecker-factored Hessian approximation, the quantization loss depends jointly on both the activation and gradient covariances, and KronQ exploits this at two complementary levels. (1) KronQ introduces bidirectional incoherence processing, extending the existing input-side random rotation to the output dimension using the gradient covariance, reducing weight magnitude variance across both input and output dimensions. (2) KronQ derives a new sensitivity metric for inter-layer mixed-precision allocation, driven by the gradient and activation Hessian traces. Notably, in the case of 2-bit weight-only quantization on LLaMA-3-70B, while GPTQ and GPTAQ diverge or produce degenerate quantizations (>2000 perplexity on WikiText-2), KronQ achieves 7.93 perplexity.

# KronQ: 通过 Kronecker 因子化 Hessian 的 LLM 量化方法 — 论文总结

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：现有的后训练量化（PTQ）方法（如 GPTQ）在构建量化目标时仅利用输入激活统计量（H_X），隐式假设所有输出通道对重建损失的贡献相同。这忽略了输出侧的梯度协方差信息，导致在极低位宽（尤其是 2-bit）量化时出现严重退化甚至发散（例如 LLaMA-3-70B 上困惑度超过 2000）。
- **背景**：LLM 参数规模持续增长，量化是减少存储和推理计算的关键技术。补偿类 PTQ（基于 OBS）和旋转类 PTQ 是主流框架，但补偿类方法仅使用激活协方差作为 Hessian 代理。Kronecker 因子化近似（K-FAC）表明完整 Hessian 可分解为 H ≈ H_X ⊗ H_G，其中 H_G 是梯度协方差。现有方法忽略 H_G，造成输出侧信息丢失。
- **整体意义**：首次系统地将梯度协方差引入 PTQ 流水线，通过双向非相干处理和联合 Hessian 迹的混合精度分配，在极低位宽下显著提升量化质量，证明了输出侧统计量的关键作用。

## 2. 论文提出的方法论

### 核心思想
- 基于 K-FAC 近似 H ≈ H_X ⊗ H_G，保留梯度协方差 H_G，修正传统方法中隐式设 H_G = I 的缺陷。
- H_G 通过单次反向传播在量化前预计算，存储为 d_out × d_out 矩阵。

### 关键技术细节
1. **Kronecker 因子化量化误差目标**：
   - 扩展逐层量化目标为：
     \[
     \min_{\hat{W}} \mathrm{tr}\left[ \mathbf{H}_G \left( \Delta \mathbf{W} \mathbf{H}_X \Delta \mathbf{W}^\top - \mathbf{W} \Delta \mathbf{X} \mathbf{X}^\top \Delta \mathbf{W}^\top \right) \right]
     \]
   - 其中包含 GPTAQ 的输入漂移校正项。

2. **双向非相干处理 (BiIP)**：
   - **输入侧**：应用随机正交变换（Hadamard）到权重和 H_X，并加入对角线缩放 S_X。
   - **输出侧**：新增输出侧对角线缩放 S_G（基于 H_G 对角线），并对权重施加与 S_G 相关的行方向缩放，再进行输出侧正交变换。
   - 处理后的权重在两个维度上的变异系数（CV）均显著降低（LLaMA-3-70B 中 CV_in 从 9.21 降至 0.29，CV_out 从 0.93 降至 0.24）。
   - **关键性质**：H_G 在列式 OBS 更新中代数消去（Proposition 1），因此 BiIP 完成后可释放 H_G，后续量化循环的内存与 GPTAQ 相同。目标函数在正交变换下不变（Theorem 1）。

3. **混合精度敏感性指标**：
   - 传统方法使用 tr(H_X) 作为敏感性分数，但同一 attention 块的 Q、K、V 层共享输入，导致无法区分。
   - KronQ 使用联合分数 s = tr(H_G) · tr(H_X)，通过 H_G 打破退化。
   - 分配较高位宽给 s 较大的子层。

### 算法流程（文字说明）
1. **预计算**：在量化前，用校准集执行一次反向传播，收集每层输出梯度，计算梯度协方差 H_G 并存储。
2. **BiIP 预处理**：
   - 计算输入协方差 H_X 和漂移项 ΔXX^⊤。
   - 计算输入侧缩放 S_X 和输出侧缩放 S_G。
   - 应用缩放：W ← S_G W S_X，并相应更新 H_X、ΔXX^⊤、H_G。
   - 应用 Hadamard 变换：W ← U W V^⊤，H_X ← V H_X V^⊤，ΔXX^⊤ ← V ΔXX^⊤ V^⊤，H_G ← U H_G U^⊤。
   - 释放 H_G（后续不再需要）。
3. **量化循环**：基于 GPTAQ 的列式 OBS 更新，按块处理权重，执行量化、补偿、漂移校正。H_G 不在循环中使用。
4. **后处理**：撤销缩放和正交变换：Q ← U^⊤ Q V；Q ← S_G^{-1} Q S_X^{-1}。
5. **混合精度分配**：根据 s = tr(H_G) · tr(H_X) 排序子层，在量化前设定位宽，然后使用上述流程逐层量化。

- 将梯度协方差 H_G 引入 PTQ 流水线能显著改善极低位宽量化质量，尤其在 2-bit 下，H_G 是关键因素。
