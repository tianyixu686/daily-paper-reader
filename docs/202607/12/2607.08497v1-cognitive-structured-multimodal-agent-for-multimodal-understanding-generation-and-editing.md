## 摘要
最近统一的多模态模型表明，单一架构可以同时执行视觉/语言理解和图像生成/编辑。然而，它们反复将所有历史视觉和文本输入馈送到共享上下文窗口中，由于视觉标记爆炸和不可靠的跨轮次引用，限制了长程多模态对话。我们提出了一种认知结构化的多模态智能体，将视觉信息外部化到情景视觉记忆（Episodic Visual Memory）中，并在推理过程中选择性地重新激活相关情景。该智能体由感知抽象引擎（Perceptual Abstraction Engine）用于结构化视觉抽象，认知检索引擎（Cognitive Retrieval Engine）用于跨轮次记忆检索，以及多模态执行控制器（Multimodal Executive Controller）用于自主任务推断和动作规划。为了解决现有数据集中缺乏轮次级检索监督的问题，我们开发了一个统一场景引擎（Unified Scenario Engine），通过编程方式生成具有细粒度检索注释的结构化多轮对话，使得强化学习能够优化抽象和检索策略。我们还构建了一个按难度分层的长程视觉对话基准来评估情景视觉回忆。我们的8B智能体在20轮会话中达到了91.4%的检索准确率，超过了32B基线8.2%，同时每轮推理时间几乎减半（23.1s -> 12.7s）。我们进一步提出了认知结构化的多模态智能体套件（CMA-Harness），这是一种工具增强的部署，具有相同的认知结构，集成了持久多模态记忆、网络访问、图像生成/编辑/合成工具以及兼容OpenAI的服务。结构化记忆和模块化决策为长程多模态智能体提供了一种比单一参数扩展更具可扩展性和效率的范式。代码：https://github.com/caseclose/cma-harness；项目页面：https://caseclose.github.io/cma-harness/

## Abstract
Recent unified multimodal models show a single architecture can jointly perform vision/language understanding and image generation/editing. However, they repeatedly feed all historical visual and textual inputs into a shared context window, limiting long-horizon multimodal dialogue due to visual token explosion and unreliable cross-turn referencing. We propose a Cognitive-structured Multimodal Agent that externalizes visual information into an Episodic Visual Memory and selectively reactivates relevant episodes during reasoning. The agent consists of a Perceptual Abstraction Engine for structured visual abstraction, a Cognitive Retrieval Engine for cross-turn memory retrieval, and a Multimodal Executive Controller for autonomous task inference and action planning. To address the lack of turn-level retrieval supervision in existing datasets, we develop a Unified Scenario Engine that programmatically generates structured multi-turn conversations with fine-grained retrieval annotations, enabling reinforcement learning to optimize abstraction and retrieval policies. We also construct a long-horizon visual-dialogue benchmark stratified by difficulty to evaluate episodic visual recall. Our 8B agent achieves 91.4% retrieval accuracy over 20-turn sessions, surpassing 32B baselines by +8.2% while nearly halving per-turn inference time (23.1s -> 12.7s). We further present the Cognitive-structured Multimodal Agent Harness (CMA-Harness), a tool-augmented deployment of the same cognitive structure integrating persistent multimodal memory, web access, image generation/editing/composition tools, and OpenAI-compatible serving. Structured memory and modular decision-making offer a more scalable, efficient paradigm for long-horizon multimodal agents than monolithic parameter scaling. ## 1. 核心问题与整体含义
现有统一多模态模型（如BAGEL、Chameleon等）将视觉和语言理解、图像生成/编辑合并到一个共享参数空间。然而在长历程多模态对话中，它们反复将所有历史视觉和文本输入塞入上下文窗口，导致两大问题：
- **视觉令牌爆炸**：图像令牌远多于文本，随着对话轮次增长，上下文预算被快速挤占。
- **跨轮视觉引用不可靠**：依赖隐式注意力机制，长距离视觉引用易产生检索错误和语义漂移。

论文提出**认知结构化的多模态智能体（Cognitive-structured Multimodal Agent）**，将视觉信息外部化到结构化情景记忆库中，仅在需要时选择性激活相关记忆，从而避免上述问题，为长程多模态交互提供更可扩展、高效的范式。

## 2. 方法论：核心思想与关键技术细节
核心思想是将长程多模态任务分解为三个独立可训练、可升级的模块，配合外部情景视觉记忆（EVM），实现按需检索。

### 2.1 三大模块
- **感知抽象引擎（PAE）**：将每张输入图像压缩为结构化记忆条目（标签、详细描述、压缩缩略图），存入EVM。
- **认知检索引擎（CoRE）**：给定当前查询和对话状态，从EVM中检索任务相关的视觉情节索引。
- **多模态执行控制器（MEC）**：进行任务分类（生成、编辑、理解/聊天、组合等），整合检索到的记忆与当前查询，调度下游模型（如Qwen-Image-Edit）执行。

### 2.2 关键技术
- **问题形式化**：将多轮对话建模为学习一个选择策略 π，从历史图像集 I<i 中选出最小相关子集 I*_i，然后基于该子集生成响应。目标是极小化 |I*_i| 且保持性能。
- **统一场景引擎（Unified Scenario Engine）**：为了弥补现有数据集缺乏轮次级检索标注，程序化生成20轮对话，包含55个主题、8个领域。每轮自动生成用户查询、真实检索集、难度标签。采用“人类模拟（Gemini）+ 零样本多智能体（Qwen-VL + Qwen-Image-Edit） + GT校验”闭环，确保标注质量。
- **难度分层与硬负例**：将对话轮次按主题切换、时间跨度、多图交互、模糊引用等分为 easy / medium / hard / very hard 四级，并注入高相似度混淆图和无须检索的负例。
- **分阶段SFT+RL训练**：
  1. SFT训练CoRE（检索器）：基于场景引擎的监督数据。
  2. RL（DAPO）优化CoRE：奖励为难度加权Jaccard相似度，强调困难轮次和负例。
  3. RL（DAPO）优化PAE（记忆写入器）：冻结CoRE，以检索准确率作为奖励，使记忆表示直接服务于下游检索。
- **MEC动作空间**：5种模式（生成、编辑、组合、理解、纯聊天），在CMA-Harness扩展为17个工具（包括Web搜索、图像抓取、背景移除等）。

## 6. 主要结论与发现
- **检索优势显著**：8B智能体在20轮英文对话中达到91.4%检索准确率，超过最强32B基线8.2%；且随难度增大优势扩大（Full→Hard差距从9.6%升至18.4%）。
- **生成质量同步提升**：Ours取得最高总分（8.49 EN / 8.53 CN），检索准确率与生成质量高度相关。
- **效率翻倍**：比32B全上下文基线推理时间减半（12.7s vs 23.1s），因只处理检索到的记忆。
- **分阶段SFT+RL有效**：每阶段带来可衡量的提升，尤其RL优化PAE使Hard准确率再提4.6%。
- **文本记忆不足**：纯文本记忆在Hard子集仅24.4%～39.0%，而完整EVM（含缩略图）达82.0%，证明了视觉细节不可缺少。
- **模块化优于参数扩展**：相同架构的8B+优化超越32B无优化，表明结构化记忆和模块化决策比单纯扩大模型更高效。

## 7. 优点
- **打破统一模型瓶颈**：首次系统性地将长程多模态对话中的视觉记忆外部化，解决了令牌爆炸和语义漂移。
- **模块化设计**：PAE、CoRE、MEC可独立训练、替换、扩展，便于性能-延迟权衡。
- **数据生成创新**：统一场景引擎提供带检索标注的合成数据，解决了真实数据缺失问题；难度分层和硬负例增加训练鲁棒性。
- **闭环优化**：RL直接以检索准确率奖励记忆写入器，实现端到端优化记忆表示。
- **CMA-Harness部署**：将相同认知结构延展到实际应用（工具调用、持久记忆、Web搜索等），证明了方法的通用性。

## 8. 不足与局限
- **基准轮次限制**：当前仅评估20轮对话，更长的交互（如50轮以上）中记忆和检索的稳定性未验证。
- **依赖特定模型族**：骨干模型（Qwen3-VL-8B）和生成模型（Qwen-Image-Edit）均为腾讯开源系列，泛化到其他基座模型（如LLaVA、MiniGPT）的效果未知。
- **训练资源未公开**：缺少GPU型号、数量、训练时间等关键信息，不利于复现。
- **合成数据偏差**：训练和评估数据均由程序生成，其多样性和与真实对话的分布差异可能影响结论的普适性。
