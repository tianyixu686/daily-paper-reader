## 摘要
多模态学习通常需要为每种模态配备专用的编码器。当涉及表格模态时，先前的研究大多使用普通MLP作为编码器。然而，如果它是一个强大的编码器，表格领域就不会成为“深度学习最后未被征服的城堡”。本研究首次在图像-表格设置中评估了最先进的表格模型作为编码器的性能。一个障碍凸显出来：在表格领域中性能最佳的方法之一——上下文学习模型，需要标签来处理实例，这使得以相同方式嵌入训练和测试实例变得不简单。我们针对该家族的多个模型解决了这一问题。通过这项研究，我们希望强调编码器因素在多模态学习中的重要性。

## Abstract
Multimodal learning usually requires a dedicated encoder per modality. When a tabular modality is involved, prior work has been mostly using a \emph{plain MLP} as the encoder. Yet if it were a strong encoder, the tabular domain would not be ``the last unconquered castle for deep learning''. This study evaluates state-of-the-art tabular models as encoders in the image-tabular setting for the first time. An obstacle stands out. In-Context Learning models, among the best performing methods in the tabular domain, require labels to process instances, making it non-trivial to embed training and test instances the same way. We addressed this problem across multiple models of this family. With this study, we would like to highlight the importance of encoder factor in the multimodal learning.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **问题背景**：多模态学习通常需要为每种模态配备独立编码器。在图像-表格多模态设置中，先前研究几乎普遍采用简单的浅层MLP作为表格编码器。然而，表格领域被称为“深度学习最后未被征服的城堡”，说明现有的MLP编码器性能不足，亟需更强的候选方案。
- **核心动机**：更强大的表格预测器也应能生成更好的表格编码表示，从而提升多模态融合效果。因此，有必要系统评估最先进的表格模型（包括In-Context Learning表格基础模型）作为多模态编码器的潜力。
- **关键障碍**：ICL TFM（如TabPFNv2）需要标签来区分上下文和查询角色，导致同一实例在训练（上下文）和测试（查询）角色下产生不同的表示（即“上下文-查询表示偏移”），这在下游任务中违反了i.i.d.假设。该偏移是否普遍存在于其他ICL模型，以及能否缓解，尚未有研究。

  3. **ICL特征提取方案**（解决表示偏移）：
     - **Vanilla (⋆)**：训练实例以上下文角色提取，测试实例以查询角色提取（导致偏移）。
     - **Leave-one-fold-out (†)**：将训练集分成K折，每折作为查询，其余作为上下文，使每个训练实例都曾被编码为查询。

## 6. 论文的主要结论与发现

1. **排榜不稳定**：多模态方法排名随表格编码器显著变化，从单一编码器得出的结论不能推广。
2. **融合可能退化**：在模态主导的数据集中（如图像主导的CCD、表格主导的Petfinder），融合反而导致性能低于单模态基线，该现象不限于双线性融合（TIP也有）。
3. **多模态提升部分源于编码器补偿**：OLS回归显示，随着表格编码器质量提升，融合带来的相对增益减小（斜率 < 1），表明部分增益是弥补弱编码器的不足，而非真正的跨模态交互。
4. **简单基线可匹敌复杂方法**：在“可能多模态”数据集上，双线性融合+强ICL编码器（如TabPFN ≀）性能与精心设计的TIP方法相当，但参数少13.2倍且无需预训练。
5. **表示偏移普遍存在**：TabPFNv2、TabDPT、TabICLv2均存在明显偏移；Vanilla提取一致劣于LOFO和NP；NP是推荐方案（多数情况下优于LOFO）。
6. **ICL模型作为编码器需谨慎处理**：必须使用NP或LOFO缓解偏移，否则性能严重下降。

## 7. 优点：方法或实验设计上的亮点

- **首次系统评估**：首次将最先进表格模型作为多模态编码器进行公平比较，填补空白。
- **解决了关键工程问题**：针对ICL模型表示偏移，提出了三种特征提取方案并给出了明确推荐（NP）。
- **设计公正的评估指标**：RPR消除了数据集间尺度差异，EDS量化了分布偏移，使比较更客观。

- **深入分析机制**：通过OLS回归揭示多模态提升的源头，区分真正的跨模态交互与编码器补偿。
