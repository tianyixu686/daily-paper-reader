## 摘要
最近的多模态大语言模型（MLLMs）在“用图像思考”（Thinking with Images, TwI）范式下，通过迭代执行各种视觉工具操作，在细粒度感知任务上取得了显著进展。然而，这种范式严重依赖频繁的外部工具调用和重复的图像重新编码，导致大量计算开销和推理延迟。为了解决这些问题，我们提出了“超越眼睛”（Beyond the Eye, BEE），一种以自我调节能力为核心的新型隐式视觉工具范式。BEE直接将视觉工具调用行为纳入训练目标，并鼓励模型发展自我调节的调用机制。这种设计使模型能够自适应地平衡内部知识和隐式工具，避免冗余的工具使用，同时大幅减少推理延迟。具体来说，BEE包含两个阶段的训练过程：（1）形式化的思维链（CoT）监督微调。我们构建具有结构化工具槽位和混合调用状态的CoT轨迹。这一阶段激活模型的隐式工具表征和自适应切换能力。（2）自我调节的奖励驱动对齐。为了解决由模糊认知边界引起的冗余工具使用问题，我们首先引入净工具增益（NTG）指标来量化该现象。基于这一观察，我们进一步提出了一种自我调节的奖励机制。该机制惩罚无效的工具依赖，并鼓励模型进行知识路由，确保仅在模型内部知识不足时才调用隐式工具。BEE在细粒度视觉感知方面取得了最先进的性能，同时在通用推理任务中保持竞争力，并在推理效率方面获得了显著提升。

## Abstract
Recent multimodal large language models (MLLMs) have made remarkable progress on fine-grained perception tasks under the "Thinking with Images" (TwI) paradigm by iteratively performing various visual tool operations. However, this paradigm relies heavily on frequent external tool calls and repeated image re-encoding, which leads to substantial computational overhead and inference latency. To address these issues, we propose Beyond the Eye (BEE), a novel implicit visual tool paradigm centered on self-regulated capability. BEE directly incorporates visual tool invocation behaviors into the training objective and encourages the model to develop a self-regulated invocation mechanism. This design enables the model to adaptively balance internal knowledge and implicit tools, avoiding redundant tool usage while substantially reducing inference latency. Specifically, BEE involves a two-stage training process: (1) Formalized Chain-of-Thought (CoT) Supervised Fine-tuning (SFT). We construct CoT trajectories with structured tool slots and mixed invocation states. This stage activates the model's implicit tool representations and adaptive switching capability. (2) Self-regulated Reward-Driven Alignment. To address redundant tool usage caused by ambiguous cognitive boundaries, we first introduce the Net Tool Gain (NTG) metric to quantify this phenomenon. Based on this observation, we further propose a self-regulated reward mechanism. This mechanism penalizes ineffective tool dependency and encourages the model to perform knowledge routing, ensuring that implicit tools are invoked only when the model's internal knowledge is insufficient. BEE achieves state-of-the-art performance in fine-grained visual perception while remaining competitive in general reasoning tasks and achieving substantial gains in inference efficiency.

- **整体含义**：为解决上述问题，论文提出 **BEE（Beyond the Eye）**，一种基于**自我调节（self-regulated）** 能力的**隐式视觉工具（Implicit Visual Tool）** 范式。BEE将工具调用行为内化到模型训练目标中，使模型学会在自身知识储备不足时才激活隐式工具，从而在保持甚至提升感知精度的同时，大幅降低推理延迟，实现更高效、更自适应的多模态推理。

## 2. 方法论

- **核心思想**：将外部工具操作（如裁剪、旋转、对比度增强等）转化为**隐式视觉特征令牌（implicit tool tokens）**，通过自回归生成内嵌到推理过程中，不再依赖外部API和重复图像编码。同时，设计一个自我调节机制，让模型根据当前能力和问题难度决定是否需要激活这些隐式工具。
- **关键技术细节**：
  - **两阶段训练流程**：
    1. **阶段1：形式化CoT监督微调（Formalized CoT Supervised Fine-tuning）**  
       - 使用预构建的形式化CoT标签数据（包含带有隐式工具槽位的推理轨迹和纯文本推理轨迹），训练模型激活隐式工具表征和自适应切换能力。  
       - 数据来源：融合Refocus、Thyme、mini-o3、Visual-CoT等异构数据集，经过代码沙盒生成、质量过滤、语义编码转换和文本改写，最终形成BEE-SFT-320K数据集。  
       - 隐式工具令牌由预训练的**语义工具分词器（Semantic Tool Tokenizer）** 将目标图像（如裁剪后的局部区域）编码为离散代码令牌。
    2. **阶段2：自我调节奖励驱动对齐（Self-regulated Reward-driven Alignment）**  
       - 针对阶段1模型仍存在冗余工具调用的问题，引入**净工具增益（Net Tool Gain, NTG）** 度量，量化工具调用的实际边际收益。  
       - 提出**自我调节奖励机制（MC-Reward）**，包含四个子奖励：答案正确性（\(r_{acc}\)）、推理-答案一致性（\(r_{cons}\)）、格式符合性（\(r_{fmt}\)）、自我调节置信度校准（\(r_{mc}\)）。  
       - \(r_{mc}\) 根据模型在rollout阶段的群体平均正确率 \(p\) 动态调整：当 \(p\) 低于阈值 \(\tau\)（设为2/3）时，鼓励使用工具；当 \(p \ge \tau\) 时，惩罚不必要的工具调用。使用连续平滑函数（线性基线+余弦门控）避免奖励不连续。
  - **优化算法**：使用DAPO（Decoupled Alignment from Policy Optimization）进行强化学习优化，目标函数为标准带裁剪的策略梯度。
- **算法流程**（文字说明）：  
  1. 用BEE-SFT-320K数据集对基础模型（如Qwen2.5-VL-7B/ Qwen3-VL-4/8B）进行SFT训练。  
  2. 在BEE-RL-55K数据集上，使用DAPO算法和组合奖励函数（含MC-Reward）进行强化学习对齐，模型通过自采样生成多条轨迹，基于奖励排名计算优势，更新策略。

## 6. 主要结论与发现

- BEE在细粒度视觉感知任务上达到SOTA：BEE-8B平均75.3%准确率，超越106-235B开源模型及Kimi-K2.6等。  
- 在通用推理和幻觉检测上也优于基础模型，表明提升不是过拟合特定感知任务。  
- 推理效率大幅提升：与Thyme相比，8K分辨率下加速约86.2%；吞吐量从0.07 sample/s提升至1.61 sample/s。  
- 自我调节奖励机制（MC-Reward）有效抑制冗余工具调用，使NTG从6.2%提升至66.0%，同时降低延迟和成本。  
- 两阶段训练策略成功赋予模型自适应工具路由能力——简单问题少用工具，困难问题适当调用隐式工具。

## 7. 优点

- **端到端联合优化**：隐式工具生成与推理在同一前向过程完成，消除外部工具调用的分离和级联误差。  
- **推理高效**：单次前向推理即可完成工具增强推理，无需重复图像编码和I/O操作。  
- **能力自适应触发**：模型可根据当前能力边界动态决定是否使用工具，避免简单问题上浪费计算，复杂问题上提供必要帮助。  
- **数据高效训练**：相比DeepEyesV2和Thyme，BEE使用更少数据和更短训练时间达到更优性能。  
- **部署友好**：推理仅需标准MLLM部署，不依赖外部沙箱或额外代理，成本更低（每千样本0.72美元 vs DeepEyes 6.4美元）。  
- **理论分析**：给出MC-Reward的参数约束和连续奖励设计的数学推导，确保训练稳定性。

## 8. 不足与局限

- **逻辑推理能力有限**：在VisuLogic、MathVerse等复杂推理基准上表现较弱，有时跳过详细推理步骤直接给出答案。原因在于训练数据中缺乏复杂推理示例。  
- **多物体场景遗漏**：在自动驾驶等包含大量视觉元素的场景中，模型可能忽略部分目标（如少数车辆），可能与训练数据中密集多物体感知实例不足有关。  
- **隐式工具表达能力**：当前隐式令牌主要覆盖裁剪、旋转、对比度增强等基础操作，尚未包含更复杂的视觉推理（如空间关系推理、细粒度属性检测），工具集有待扩展。  
- **阈值τ固定**：论文将τ设为2/3，虽经消融验证其有效性，但固定阈值可能不完美适应所有模型和能力变化阶段，未来可考虑动态自适应调整。  
- **部分评估依赖裁判模型**：答案正确性和一致性使用Qwen3-VL-235B或Qwen3-VL-4B作为裁判，可能引入裁判自身偏差。  
- **未公开全部训练数据**：虽然代码已开源，但训练数据的完整复现细节（如人工清理过程）可能不易完全重现。
