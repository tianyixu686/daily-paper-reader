## 摘要
交替进行推理和工具使用的语言智能体，随着推理链的延长，性能会急剧下降，即使每个单独的步骤都很简单。我们将其归因于上下文稀释：智能体的调查状态（已确认的内容、怀疑的内容以及仍需探究的内容）仅隐式地存在于不断增长的上下文窗口中，早期的发现被后续的检索所埋没。我们引入了SLEUTH，通过结构化的元认知工作记忆使这种状态显式化并可操作：智能体维护已确认的事实（源自来源）、按证据排序的活跃假设以及直接驱动其下一步行动的未解问题。在五个多跳基准测试和五个已建立的基线上，SLEUTH的优势随难度增加而增长，从HotpotQA上的+5分到4跳链上的+11分，超越了无需多次情节的Reflexion。在分析剩余差距所在时，我们识别出证据充分性问题：智能体通常能找到答案但未能确定，将其预算浪费在不必要的验证上。一个轻量级的承诺触发器解决了这个问题，但仅当智能体已经维护了结构化状态时才有效：相同的触发器应用于非结构化智能体时没有改善，从而隔离出有组织的认识状态作为有效承诺的必要条件。最后，在较弱模型上强制执行协议遵守，在最困难的问题上恢复了高达+19分，表明智能体如何组织其推理（而非原始模型能力）是扩展多跳推理的主动要素。

## Abstract
Language agents that interleave reasoning and tool use degrade sharply as reasoning chains lengthen, even when each individual step is easy. We trace this to context dilution: an agent's investigative state (what it has confirmed, what it suspects, and what it still needs) lives only implicitly in a growing context window, where early discoveries are buried under later retrievals. We introduce SLEUTH, which makes this state explicit and actionable through a structured epistemic working memory: the agent maintains Confirmed Facts grounded to sources, Active Hypotheses ranked by evidence, and Open Questions that directly drive its next action. Across five multi-hop benchmarks and five established baselines, SLEUTH's advantage grows with difficulty, from +5 points on HotpotQA to +11 on 4-hop chains, surpassing Reflexion without multiple episodes. Analyzing where the remaining gap lies, we identify the evidence sufficiency problem: agents often find the answer but fail to commit, exhausting their budget on needless verification. A lightweight commitment trigger fixes this, but only when the agent already maintains structured state: the identical trigger applied to an unstructured agent yields no improvement, isolating organized epistemic state as the necessary condition for effective commitment. Finally, enforcing protocol adherence on a weaker model recovers up to +19 points on the hardest problems, showing that how an agent organizes its reasoning, not raw model capability, is the active ingredient for scaling multi-hop reasoning.

## 2. 方法论：核心思想、关键技术细节、流程

### 2.1 核心思想

引入**结构化元认知工作记忆**（Structured Epistemic Working Memory），将智能体的推理状态显式化、可操作化，使智能体在每个步骤后输出并更新三部分结构：

- **已确认事实（Confirmed Facts）**：从工具输出中提取的已验证陈述，每个事实附带来源（段落标题）。单调累积，不撤回。
- **活跃假设（Active Hypotheses）**：候选答案，按证据支持排序，每个假设关联支持/矛盾的事实ID及置信度（高/中/低）。
- **未解问题（Open Questions）**：仍需调查的不确定性，每个问题附带建议行动（搜索/查找）和优先级。最高优先级的未解问题直接驱动智能体的下一步工具调用。

### 2.2 关键技术细节

- **更新协议**（每轮工具调用后）：
  1. 提取新事实并加入已确认事实（证据单调性）。
  2. 更新假设（调整置信度、生成新假设）。
  3. 更新未解问题（移除已解答问题、生成新问题）。
  4. 执行最高优先级未解问题的建议动作；若未解问题为空且某假设置信度高，则调用 `finish(answer)`。
- **承诺机制（Nudge）**：
  - 由于上述结构没有明确停止标准，智能体会过度验证（证据充分但不确定答案），耗尽预算。
  - 引入轻量级触发器：当 **已用轮次 ≥ 总预算的 α（默认70%）** 且 **已确认事实数 ≥ τ（默认2）** 时，强制智能体从已确认事实和活跃假设中综合答案，无需进一步搜索。α=0.7 控制探索-利用权衡。
- **实现**：完全通过提示工程（系统提示）实现，无需微调或架构修改。对于指令遵循能力弱的模型（如GLM-5），增加格式约束和运行时检查（拒绝非结构响应并重新提示），使协议遵守率从~15%提升至~99%。

### 2.3 算法流程（文字说明）

```
每轮 t：
  1. 观察工具返回结果。
  2. 更新工作记忆 W_t = (F_t, H_t, Q_t)：
     - 提取新事实加入 F_t
     - 更新 H_t 的置信度
     - 更新 Q_t（移除已解答、生成新问题）
  3. 选择动作：
     - 若 Q_t 非空：执行最高优先级问题对应的动作
     - 若 Q_t 为空且某假设置信度高，调用 finish(answer)
  4. 若满足 nudge 条件（t ≥ αT 且 |F_t| ≥ τ），发送合成消息强制综合答案。
```

## 6. 主要结论与发现

1. **SLEUTH在所有基准上达到最佳**：比最强外部基线提升+2.8至+11.9分，且优势随难度增加而扩大（HotpotQA +4.7 → 4跳MuSiQue +10.9）。
2. **结构化工作记忆是有效承诺的必要条件**：相同的nudge触发器应用于ReAct无效果，只有结合SLEUTH的结构化状态才能提升+4分。
3. **证据充分性问题**：智能体找到答案但无法承诺（无nudge时超时率24.7%）；92%的超时情节历史中包含正确答案。nudge解决此问题，但依赖于结构化状态。
4. **推理组织比模型能力更重要**：较弱模型（GLM-5）被强制遵守协议后（+19分）甚至超过较强模型（Sonnet）的ReAct（42.0%），表明组织方式是主动要素。
5. **两个核心失效模式**：
   - **信息丢失**（浅层问题）：结构化事实跟踪防止证据被掩埋。
   - **信息瘫痪**（深层问题）：nudge机制强制利用证据，避免过度验证。
6. **协议遵守至关重要**：GLM-5遵守率从15%提到99%，4跳EM从29.9%升至48.9%。

## 8. 不足与局限

- **单调事实策略**：已确认事实永不撤回，可能导致早期错误固化（例如技术上正确但上下文中误导的事实）。论文给出了一个“旧金山 vs 萨克拉门托”的具体失败案例。
