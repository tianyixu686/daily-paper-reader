## 摘要
智能体检索增强生成（Agentic RAG）通过允许语言模型迭代推理、生成搜索查询、检索证据和预测答案，扩展了静态RAG。然而，模型在决定何时检索、是否使用词汇匹配或语义相似性，以及如何控制上下文粒度以防止无关标记干扰智能体推理方面仍面临挑战。本文提出了GRASP，一个用于训练智能体在多步推理过程中自适应协调互补检索工具的强化学习框架。GRASP为智能体提供了语义搜索、关键词搜索和段落阅读动作，使其能够在需要时检索句子级证据并扩展更多上下文。我们使用一个联合考虑答案准确性、基于证据的阅读、互补搜索和轮次效率的奖励来训练策略。在多跳推理基准上的实验表明，与单步检索、基于提示的智能体RAG和基于强化学习的检索基线相比，GRASP提高了检索召回率和下游问答性能。定性和消融分析显示，学习到的策略形成了可解释的略读和扫描行为：它使用语义搜索进行广泛探索，段落阅读进行局部验证，关键词搜索寻找实体特定证据。这些结果表明，学习协调检索信号和上下文粒度对于智能体的正确推理至关重要。

## Abstract
Agentic retrieval-augmented generation (RAG) extends static RAG by allowing language models to iteratively reason, generate search queries, retrieve evidence, and predict answers. However, it remains challenging for models to decide when to retrieve, whether to use lexical matching or semantic similarity, and how to control context granularity to prevent irrelevant tokens from interfering with agent reasoning. In this paper, we introduce GRASP, a reinforcement learning (RL) framework for training agents to adaptively coordinate complementary retrieval tools during multi-step reasoning. GRASP provides the agent with semantic search, keyword search, and paragraph-reading actions, enabling it to retrieve sentence-level evidence and expand further context only when needed. We train the policy with a reward that jointly accounts for answer accuracy, grounded reading, complementary search, and turn efficiency. Experiments on multi-hop reasoning benchmarks show that GRASP improves both retrieval recall and downstream question answering performance compared with single-step retrieval, prompting-based agentic RAG, and RL-based retrieval baselines. Qualitative and ablation analyses show that the learned policy develops interpretable skimming and scanning behavior: it uses semantic search for broad exploration, paragraph reading for local verification, and keyword search for entity-specific evidence. These results suggest that learning to coordinate retrieval signals and context granularity is critical for agent's correct reasoning.

  1. **何时检索**：难以判断当前推理状态是否需要新的外部证据。
  2. **使用何种检索信号**：无法自适应地切换**词汇匹配**（如 BM25）与**语义相似度**（如密集检索），而不同推理阶段需要不同的信号（例如实体查找需要关键词匹配，概念探索需要语义搜索）。
  3. **如何控制上下文粒度**：粗粒度的段落检索会引入大量无关噪声，干扰推理链；细粒度的句子检索又可能丢失必要的上下文。现有方法缺乏动态调整检索粒度的能力。
- **研究动机**：现有静态 RAG 或简单多步 RAG 往往使用单一检索器（要么仅语义，要么仅词汇）和固定的段落级粒度，导致在多跳复杂推理中易产生幻觉或遗漏证据。作者认为，**学习协调互补检索工具并控制上下文粒度**，是智能体正确推理的关键。

## 二、方法论：核心思想、关键技术细节

### 2.1 核心思想

- 将智能体 RAG 建模为**有限时域马尔可夫决策过程 (MDP)**，状态包含查询、交互历史、已检索证据、剩余回合数；动作空间为四种工具调用。
- 策略模型（LLM）通过强化学习（GRPO）训练，学习何时、使用何种工具、以何种粒度检索，最终生成答案。

### 2.2 动作设计

- 定义三个证据获取工具 + 一个终止动作：
  - **语义搜索 (τs)**：基于密集向量检索，用于概念/主题探索，返回 top‑K 句子及所属段落 ID。
  - **关键词搜索 (τk)**：基于词汇匹配（BM25），用于精确实体/术语查找，返回 top‑K 句子。
  - **段落阅读 (τr)**：根据搜索返回的句子 ID，读取其完整父段落，提供更丰富的上下文。
  - **终止 (τa)**：输出最终答案。

### 2.3 奖励设计

- 总奖励 **R = RA + α·RR + β·RS + γ·RE**，最大为 2.0（RA 上限 1.0，辅助奖励之和上限 1.0）。
  - **RA (答案准确性)**：预测答案与参考答案的 token 级 F1。
  - **RR (基于证据的阅读)**：评估段落阅读动作是否针对包含金标证据的段落（F1 衡量）。
  - **RS (互补搜索)**：若语义搜索和关键词搜索均至少检索到一条金标证据，则奖励 1。
  - **RE (轮次效率)**：正确回答时，使用回合数越少奖励越高（T − T_cur / T）。
  - 权重：α=0.7, β=0.15, γ=0.15，确保 RA 为主导。

### 2.4 学习算法

- 使用 **Group-Relative Policy Optimization (GRPO)**，针对同一查询采样 G 条轨迹，计算组内相对优势，更新策略。避免了传统 PPO 中批评家网络的需求。

### 2.5 训练细节

- 策略基于 **Qwen/Qwen2.5-3B-Instruct**，全参数微调。
- 训练数据：HotpotQA distractor 训练集（90,447 个示例），每例含 2 个金标段落 + 8 个干扰段落。
- 检索索引：句子级索引，保留父段落元数据。
- 损失掩码：工具观测返回的 token 不参与策略梯度更新。
- 最大回合数 T=10，每 prompt 采样 G=4 条轨迹，每步 64 个 prompt，共 256 条轨迹/步。

  - **HotpotQA**（distractor 配置）：训练模型，并作为域内测试集。

### 3.2 对比方法

## 六、主要结论与发现

1. **GRASP 在检索召回率和 QA 精度上均显著优于所有基线**：在三个数据集上，GRASP 的召回率均最高或接近最高（HotpotQA 0.90，2Wiki 0.90，MuSiQue 0.70）；QA 的 EM、F1、JD 均领先（如 HotpotQA EM 0.53 vs 单步混合 0.37 或 Search-R1 0.45）。
2. **学习到的策略具有可解释的浏览-扫描行为**：
   - 语义搜索用于**广泛探索**；
   - 段落阅读用于**局部验证**；
   - 关键词搜索用于**精确实体查找**。
   - 行为类似于人类信息觅食：先略读大意，再细读关键段落，最后扫描具体词汇。
3. **互补检索信号和粒度控制对正确推理至关重要**：消融表明去除语义搜索或段落阅读损害最严重（去除 τs 导致 EM 下降 0.072，去除 τr 导致下降 0.122）。
4. **强化学习训练优于单纯提示式方法**：Base（无 RL）表现远差于 GRASP，说明策略学习对工具协调是必要的。

## 七、优点

- **方法设计新颖**：首次在 Agentic RAG 框架中联合学习**检索工具选择**和**上下文粒度控制**，动作空间包含句子级检索和段落扩展。
- **奖励设计合理**：多目标奖励（准确率、证据阅读、互补搜索、效率）有效引导了理想行为，且权重设计防止了 reward hacking。
- **行为解释性强**：通过马尔可夫链分析工具使用模式，证明了策略学到了类似人类信息搜索的习惯，提高了可解释性。
- **实验完整且公平**：对比了多种基线（包括提示式和 RL 式），在多个数据集评估，消融充分，并公开了训练配置和代码（基于开源框架）。
- **模型轻量高效**：使用 3B 参数的 Qwen2.5-Instruct，训练仅需 2 张 A100，结果优于依赖 GPT‑5 mini 的 IRCoT，展示了小模型通过 RL 也能达到强性能。

## 八、不足与局限

1. **对金标注释的依赖**：
   - 奖励函数 RR 和 RS 都需要**金标支持事实段落**（ground-truth supporting paragraphs），这限制了框架仅能用于具备此类细粒度标注的数据集（如 HotpotQA）。实际应用中，大多数语料没有这种标注，需探索弱监督方案。
2. **偶尔提前终止与不完全检索**：
   - 尽管平均工具调用数稳定在约 8 次，但策略仍会在证据未充分收集时过早提交答案。论文建议未来可通过奖励塑造惩罚提前承诺。
3. **实验规模与模型尺度有限**：
   - 仅使用 3B 参数模型，未验证更大模型（如 7B、14B）下的行为变化和收敛性。策略的涌现模式（如关键词搜索切换）可能随模型规模改变。
4. **跨域泛化评估不够深入**：
   - 虽然测试集包括三种数据集，但训练只在 HotpotQA 上进行。跨域实验中，在 MuSiQue 上性能提升相对较小（EM 0.23 vs Search-R1 0.22），可能需要领域自适应训练。
5. **检索工具固定为三种**：
   - 动作空间未包含动态确定检索粒度（如自动决定返回句子还是段落）的灵活性，而是通过专门工具实现。未来可探索让智能体自行决定返回多少上下文。
6. **未讨论多轮对话或开放性问答**：
   - 实验限于单轮多跳 QA，未涉及对话式或需要多轮交互的复杂任务，实际应用中可能需扩展。
