## 摘要
多跳问答和检索增强推理需要选择联合有助于回答查询的证据段落。然而，大多数检索器仍然独立评分段落或做出局部监督的顺序决策，当证据有用性取决于段落间的兼容性时，这种方法可能失败。基于LLM的集合选择可以建模此类交互，但其计算成本限制了实际应用。我们通过将多跳检索形式化为查询-集合兼容性评分来弥合这一差距，并提出了一个集合级检索框架。我们的训练目标使检索器能够将完整且兼容的证据集排在包含噪声的不完整替代方案之上，从而使集合评分对变长和部分噪声上下文更加鲁棒。我们使用两个互补的集合评分器实例化了该框架：ParaSet，一个轻量级的后期交互评分器，在预计算的双编码器嵌入上应用自注意力以实现快速候选集探索；以及SetCE，一个基于交叉编码器的重排序器，使用相同的集合级目标进行训练。在多个多跳QA基准上的实验表明，集合级兼容性学习提高了检索性能和下游QA任务性能。我们进一步表明，所提出的集合级检索器不仅优于文档级检索器，而且展现出互补的检索特性：结合它们的输出比简单地从单个文档级检索器中检索更多段落能获得更强的性能。

## Abstract
Multi-hop question answering and retrieval-augmented reasoning require selecting evidence passages that are jointly useful for answering a query. However, most retrievers still score passages independently or make locally supervised sequential decisions, which can fail when evidence usefulness depends on compatibility among passages. LLM-based set selection can model such interactions, but its computational cost limits practical use. We address this gap by formulating multi-hop retrieval as query-set compatibility scoring and propose a set-level retrieval framework. Our training objective teaches retrievers to rank complete and compatible evidence sets above incomplete, noisy alternatives, making set scoring more robust to variable-length and partially noisy contexts. We instantiate the framework with two complementary set scorers: ParaSet, a lightweight late-interaction scorer that applies self-attention over precomputed bi-encoder embeddings for fast candidate-set exploration, and SetCE, a cross-encoder-based reranker trained with the same set-level objective. Experiments on various multi-hop QA benchmarks show that set-level compatibility learning improves retrieval performance and downstream QA task performance. We further show that the proposed set-level retrievers not only outperform document-level retrievers, but also exhibit complementary retrieval characteristics: combining their outputs yields stronger performance than simply retrieving more passages from a single document-level retriever.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：多跳问答和检索增强推理（RAG）需要选择一组共同有用的证据段落（即证据集），而非单个独立段落。然而，现有大多数检索器采用独立评分（bi-encoder/cross-encoder）或局部监督的顺序决策（如beam search中逐段分类），无法建模段落之间的兼容性：一个在孤立时有用的段落，当与不兼容的上下文结合时可能变得无用甚至误导。
- **现有方法局限**：
  - 独立检索（如DPR）假设段落间独立，忽略交互。
  - 顺序检索（如ListCE）通过局部BCE训练下一段落选择，但训练信号局限于当前前缀，未优化全局集合兼容性；且对变长、含噪声的前缀鲁棒性差。
  - LLM-based set selection（如SETR）能建模交互，但计算和内存开销巨大，难以大规模或低延迟场景。
- **整体含义**：本文提出将多跳检索重新形式化为**查询-集合兼容性评分**问题，并设计一个高效的集合级检索框架，在不依赖LLM的情况下实现集合级的交互建模与高效搜索，提升检索质量和下游QA性能。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想
- **查询-集合评分**：直接学习一个评分函数 \( S(q, S) \)，评估查询 \( q \) 与候选段落集合 \( S \) 的整体兼容性，而非独立或局部分数。
- **集合级兼容性学习**：通过一个模型无关的边际排序损失，训练评分器将完整、兼容的集合排在各种不完整、含噪声或不相容的集合之上。

### 2.2 关键技术细节

#### (1) 目标函数
- **候选集构建**：对每个查询的黄金证据集 \( G_q \) 生成结构化扰动（添加、删除、替换），并加入批内负例。
- **集合质量排序**：按**召回优先、精度其次**的词典序定义集合质量排序（例如：完整黄金集 > 黄金集+干扰 > 黄金子集 > 部分黄金+干扰 > 全干扰）。
- **损失函数**：
  \[
  \mathcal{L}_{\text{final}} = \sum_q \left( \mathcal{L}_{q}^{\text{pos-all}} + \lambda \mathcal{L}_{q}^{\text{adj}} \right),
  \]
  其中：
  - \( \mathcal{L}_{q}^{\text{pos-all}} = \sum_{L \in C\setminus\{G_q\}} \max(0, \gamma - S(q, G_q) + S(q, L)) \)：黄金集必须高于所有其他候选。
  - \( \mathcal{L}_{q}^{\text{adj}} = \sum_{(H,L) \in \mathcal{P}_{\text{adj}}} \max(0, \gamma - S(q, H) + S(q, L)) \)：相邻质量等级的集合之间也需保持边际。
- 该目标可训练任何评分架构（包括ParaSet和SetCE）。

#### (2) ParaSet：轻量级集合探索器
- **架构**：基于预计算的双编码器嵌入（query embedding \( q \)，passage embeddings \( p_i \)）。对候选集合，构造序列 \([CLS; q; p_{i_1}; ...; p_{i_m}]\)，经过单层多头自注意力（晚期交互），由[CLS]输出得到分数。
- **效率优势**：段落嵌入可预计算并缓存key/value，评分时仅需query相关的轻量计算，支持对大量候选集并行评分。
- **搜索算法**：使用**束搜索**（beam search）在候选段落池中近似寻找最优集合。束初始化为top-B单例集，每步扩展（最多加一个段落），所有扩展集被并行评分并累计入候选池 \( \mathcal{S}_{\text{acc}} \)，最终返回全局最高分集合（而非仅最终束）。

#### (3) SetCE：表达性集合重排序器
- **架构**：与ListCE相同的交叉编码器（deberta-v3-base），但训练时改用上述集合级兼容性目标（替换点式下一段落BCE损失）。推理时同样使用束搜索，但评分基于完整集合的联合编码。
- **角色**：作为精排器，对ParaSet候选出的少量集合进行更精细的评分。

#### (4) 检索-排序流水线
- **ParaSet（快速探索）** → **SetCE（精细重排序）**：类似标准检索中的双编码器+交叉编码器，但操作对象是段落集合而非独立段落。
