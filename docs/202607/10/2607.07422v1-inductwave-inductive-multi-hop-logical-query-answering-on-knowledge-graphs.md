## 摘要
知识图谱上的逻辑多跳查询回答可以表述为在隐式完备性假设下的查询。当前工作主要关注存在一阶逻辑（EFO）查询。这些EFO查询包含合取、析取和否定算子。大多数现有工作采用直推式推理，即无法对训练中未见实体进行推理。现实世界中存在资源稀缺问题，我们无法使用大型知识图谱的所有节点训练模型。因此，我们提出InductWave，一种基于小波的归纳式嵌入方法，用于大型知识图谱上的逻辑查询回答。在此，训练图的节点数少于测试图。我们的模型在仅使用一半消息传递层数的情况下，性能与基线模型相当。在大多数情况下，它使用75%的层数即可超越所有基线模型。较少的资源需求使我们能够在巨型图（如Wiki-KG）上评估InductWave。我们通过在FB15k-(237)数据集的不同训练-测试图比例上进行大量实验测试模型，并将其与最先进模型进行比较。模型的代码和数据集可在https://github.com/kracr/inductwave/获取。

## Abstract
Logical Multi-Hop Query Answering over Knowledge Graphs (KGs) can be formulated as querying, with an implicit completeness assumption. Current works mainly focus on Existential First Order Logic (EFO) queries. These EFO queries contain conjunction, disjunction, and negation operators. Most existing works employ transductive reasoning, meaning they are not capable of reasoning over entities unseen during training. In the real world, there is a resource scarcity, and we cannot train a model with all the nodes of a large KG. Hence, we propose InductWave, a wavelet-based inductive embedding method for logical query answering on large KGs. Here, the training graph consists of fewer nodes than the test graph. Our model performs on par with the baseline models while having half the number of message-passing layers. It outperforms all of them in most cases, with 75% of the layers. These fewer resource requirements enable us to evaluate InductWave on massive graphs, such as Wiki-KG. We test our model using extensive experiments across varying train-test graph proportions of the FB15k-(237) dataset, comparing it with the state-of-the-art models. The code and datasets for the model are available at https://github.com/kracr/inductwave/.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：多跳逻辑查询回答在知识图谱上通常假设完备性，现有方法多为**直推式**推理，即训练时需包含所有实体，无法泛化到训练中未见的实体。随着知识图谱规模扩大（如数百万节点），训练全部节点在资源上不可行。
- **研究动机**：需要一种**归纳式**方法，在训练图（节点更少）上学习，然后能够推理到测试图（包含新节点）上。现有归纳方法如GNN-QE和NodePiece-QE各有缺陷：GNN-QE使用内存密集型的NBF-Net，难以扩展到大图；NodePiece-QE无法处理否定操作符。
- **整体含义**：提出InductWave，一种基于小波变换的归纳嵌入方法，旨在用更少的消息传递层数达到甚至超越现有模型的性能，同时降低资源需求，使其能够扩展到百万级节点的大图（如Wiki-KG）。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：将**图谱小波嵌入**与**神经贝尔曼-福特网络**相结合，提出新的链接预测方法WAVBFNet，用于关系投影操作。通过增加结构信息，使得模型在较少消息传递层数下更快收敛。
- **关键技术细节**：
  - **知识图谱拉普拉斯矩阵**：针对有向图及多种关系，定义关系拉普拉斯矩阵（带磁拉普拉斯变体），并归一化以包含全局节点度信息。
  - **图谱小波嵌入**：对每个关系拉普拉斯矩阵进行频谱分解，采用Chebyshev多项式逼近低通热核滤波器，生成每个节点的结构小波嵌入（实部和虚部）。
  - **WAVBFNet**：将源节点模糊集、查询关系嵌入与邻居的小波嵌入通过DistMult风格的乘法结合，并引入线性组合项（带或不带小波嵌入），使用PNA聚合器进行消息传递。最后经过两层前馈网络得到模糊尾节点集。
  - **模糊操作**：合取、析取、否定分别采用元素乘、加-减-乘、1减操作。
  - **训练方法**：引入遍历Dropout随机丢弃直接查询-关系边；损失函数为二元交叉熵（无需负采样）。将查询转换为后缀表示，使用栈进行非递归执行。
- **高效实现**：扩展GE-SpMM方法，使其兼容小波嵌入，将空间复杂度从O(2b|E|d)降至O(b|V|d + |E|d)。

## 3. 实验设计

- **数据集**：
  - FB15k-237：标准知识图谱数据集，包含237种关系。用于不同比例训练-测试节点划分的实验（V_inf/V_train从106%到300%）。
  - Wiki-KG：OGB基准中的大规模KG，约250万节点、1600万三元组、512种关系。用于评估可扩展性。
- **查询结构**：14种标准查询结构（1p,2p,3p,2i,3i,pi,ip,2u,up,pni,pin,inp,2in,3in），涵盖合取、析取、否定。
- **基线方法**：
  - GNN-QE（使用NBF-Net的4层消息传递）
  - NodePiece-QE（有无GNN两种变体）
  - 边类型启发式方法（无训练）

- **评价指标**：HITS@10（主要），HITS@3（额外）。对于Wiki-KG使用HITS@100。

## 6. 主要结论与发现

- **性能优势**：InductWave（3层）在大多数比例下**平均优于GNN-QE**，尤其在训练图远小于完整图时（如300%比例）优势明显。InductWave（2层）在较高比例下与GNN-QE持平甚至更好。
- **层数效率**：用一半（2层）或75%（3层）的层数达到或超越4层GNN-QE的性能，表明小波嵌入提供了额外的结构信息，加速收敛。
- **可扩展性**：在Wiki-KG上，InductWave（2层）显著优于NodePiece-QE，达到26.8%的HITS@100，而GNN-QE由于内存限制无法运行。
- **低比例时的表现**：当训练图接近完整图时（低比例），GNN-QE略优，说明此时额外的消息传递比小波嵌入更有效。
- **消融结论**：原版InductWave优于去掉线性组合或使用ComplEx的变体，表明设计的有效性。

## 7. 优点

1. **高效性**：资源消耗低，支持大图归纳推理，突破了GNN-QE的内存瓶颈。
2. **结构信息引入**：通过图谱小波嵌入，无需额外监督即可捕获节点邻域结构，加快收敛。
3. **设计简洁**：将小波嵌入与NBF-Net结合，修改消息传递公式即可，无需复杂架构。
4. **全面评估**：在多种训练-测试比例下测试，验证了方法在极端稀疏训练数据下的鲁棒性。
5. **开源可复现**：代码和数据集已公开。

## 8. 不足与局限

1. **未与ULTRA比较**：ULTRA是另一种零样本跨图谱归纳方法，作者认为目标不同而排除比较，但可能会限制读者对全面性能的认知。
2. **否定查询处理局限**：在Wiki-KG上未测试否定查询（因数据加倍问题），因此对否定算子的归纳能力未在大图上验证。
3. **低训练比例下的性能**：当训练图相对完整时，InductWave未超越GNN-QE，表明方法并非在所有场景下最优。
4. **预处理开销**：需要一次性计算拉普拉斯矩阵和小波嵌入（约10分钟），可能对快速迭代或动态图不友好。
5. **超参数敏感**：小波相关超参数（缩放因子、g、Chebyshev阶数等）需调参，文中虽进行了网格搜索但未深入分析其敏感度。
6. **实验充分性补充**：缺少与更多归纳方法（如ULTRA）的对比，且仅在两个数据集（其中一个为FB15k-237变体）上测试，泛化到其他领域知识图谱的效果未知。
