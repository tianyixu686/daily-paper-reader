## 摘要
大语言模型（LLM）智能体需要能够从环境反馈中改进长期决策的后训练方法。然而，现有的智能体后训练流水线通常将数据策展视为固定的预处理步骤，主要关注数据增强，而忽略了过滤、精炼以及对下游失败的适应。我们提出CurateEvo，一种面向智能体后训练数据策展的失败驱动动态进化框架。CurateEvo将策展策略表示为可执行代码，并利用来自保留开发集的失败轨迹迭代重写该代码。在每个周期中，进化策略将固定原始语料库转化为监督微调数据、强化学习数据和推理时记忆库。进化过程首先通过诊断重复失败模式并相应增强、过滤或精炼数据来提高有效性，然后通过基于成本感知目标剪枝冗余或低效训练轮次来提高效率。在ACEBench-Agent、BFCL-V4和τ²-Bench上的实验，在标注和野生数据设置下均显示，CurateEvo始终优于先前的策展方法，平均得分分别提升3.2和2.7分。进一步分析表明，CurateEvo与不同的后训练配方兼容，并显著降低了策展开销。

## Abstract
Large language model (LLM) agents require post-training methods that can improve long-horizon decision making from environment feedback. However, existing agentic post-training pipelines often treat data curation as a fixed preprocessing step, focusing mainly on data augmentation while neglecting filtering, refinement, and adaptation to downstream failures. We propose CurateEvo, a failure-driven dynamic evolution framework for agentic post-training data curation. CurateEvo represents the curation strategy as executable code and iteratively rewrites it using failed trajectories from a held-out development set. At each epoch, the evolved strategy transforms a fixed raw corpus into supervised fine-tuning data, reinforcement learning data, and an inference-time memory bank. The evolution process first improves effectiveness by diagnosing recurring failure modes and augmenting, filtering, or refining data accordingly, and then improves efficiency by pruning redundant or low-utility training turns under a cost-aware objective. Experiments on ACEBench-Agent, BFCL-V4, and τ^2-Bench under both labeled and wild-data settings show that CurateEvo consistently outperforms prior curation methods, improving average scores by 3.2 and 2.7 points, respectively. Further analyses demonstrate that CurateEvo is compatible with different post-training recipes and substantially reduces curation overhead.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题背景**：大语言模型（LLM）智能体需要具备长期决策能力，后训练可以提高智能体从环境反馈中学习的能力。然而，现有智能体后训练流水线通常将数据策展（data curation）视为固定的预处理步骤，主要关注数据增强（augmentation），而忽略了过滤（filtering）、精炼（refinement）以及对下游任务故障模式的动态适应。
- **两个关键局限**：
  - **策展忽视（Curation Neglect）**：多数方法只关注增加数据量，未联合优化过滤和精炼，导致低质量数据被混入训练集。
  - **适应僵化（Adaptation Rigidity）**：策展流程固定，无法针对分布外下游应用灵活调整。
- **研究动机**：提出一种动态、故障驱动的数据策展进化框架，使得策展策略能够根据模型在开发集上的失败轨迹进行迭代优化，从而提高智能体后训练的效果和效率。

## 2. 提出的方法论

### 核心思想
- **CurateEvo** 将数据策展策略表示为可执行代码（executable code），并利用保留开发集（held-out dev set）上的失败轨迹作为反馈，迭代重写该代码。
- 每轮进化中，策展策略将固定原始语料库转换为三类资源：
  - 监督微调数据（SFT data）
  - 强化学习数据（RL data）
  - 推理时记忆库（inference-time memory bank）
- 进化过程分两步：
  1. **有效性优化（Effectiveness）**：诊断重复失败模式，通过增强、过滤或精炼数据来弥补模型能力短板。
  2. **效率优化（Efficiency）**：在成本感知目标下，剪枝冗余或低效训练轮次（训练交互回合数）。

### 关键技术细节
- **策略表示**：策展策略 ρ 用代码实现，接收原始语料 Draw，输出 \( D_{sft}^\rho, D_{rl}^\rho, M_{mem}^\rho \)。
- **反馈收集**：每轮用当前策展策略训练模型，在开发集上运行得到失败轨迹 \( T_e^- \)、表现得分 \( s_e \)、策略配置文件 \( \Sigma_e \)（数据规模、轨迹长度分布等）。
- **进化目标函数**：
  \[
  J(\rho) = P(\rho) - \lambda C(\rho)
  \]
  - \( P(\rho) \)：下游有效性（开发集性能）
  - \( C(\rho) \)：归一化的训练轮次成本
  - λ 控制效果-效率权衡（实验中λ=0.3）
- **近似优化**：通过两次有序修订（先有效性后效率）近似离散代码优化，若新策略不改善目标则回滚防止错误级联。

- **效率修订**：根据失败模式和配置，删除重复/低质量样本、压缩重复交互模式、截断过长轨迹，优先保留覆盖罕见失败的样本。
- **基础模型与训练**：使用 Qwen3-4B 作为基座，采用固定 SFT+GRPO 训练配方（LoRA 微调）。
